# Contributing to yeet-git

Thank you for your interest in contributing to yeet-git!

yeet-git is an AI-powered git CLI designed to automate and enhance git workflows. To maintain its speed and reliability, we follow a few simple guidelines.

## Documentation Index

- [**Mental Model**](./docs/architecture.md): Overview of how yeet-git works.
- [**Project Rules**](./AGENT.md): Guidelines for AI agents and maintainers.
- [**Development Guide**](./docs/development.md): How to set up and run yeet-git locally.

## Workflow

1. **Fork & Clone**
   ```bash
   git clone https://github.com/samad-scs/yeet-git.git
   cd yeet-git
   npm install
   ```

2. **Develop**
   - Make your changes in `src/`.
   - Test them using `node bin/yeet.js`.

3. **Commit**
   Use `yeet` (if installed) or follow Conventional Commits (e.g., `feat:`, `fix:`, `chore:`).

4. **Pull Request**
   - Describe your changes.
   - Mention any related issues.

Please note that this project is released with a [Code of Conduct](./CODE_OF_CONDUCT.md). By participating in this project you agree to abide by its terms.
